## New-feature validation
- `server/scenePlanner.ts` — selects the most illustration-worthy, visually distinct scenes across a whole book instead of a rigid one-image-per-page mapping.
  - `selectScenes()` reads page text + the visual bible (StoryContext), asks the LLM for distinct beats, then dedupes, ranks by importance, and caps to a ceiling.
  - Deterministic fallback (`buildFallbackScenes`) = one scene per meaningful page when the LLM is unavailable.
  - `generateScenePrompts()` turns the plan into one prompt per scene, reusing the locked visual bible.
- 23 new tests in `server/scenePlanner.test.ts` cover: count clamping, dedupe (title + description, cross-page), importance/page sanitisation, ranking, fallback paths (LLM throws / empty), max-scene ceiling, and prompt pairing.

## Decoupling invariant — upheld
OCR transcription stays decoupled from image generation. scenePlanner never touches OCR and never calls the image generator. Flow: OCR text → story bible → scene plan → prompts. The story-bible (`StoryContext`) remains the sole mediator. `generateScenePrompts` returns prompts only; the pipeline is still the only caller of `generateImage`.
