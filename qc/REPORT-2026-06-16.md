## Regressions found & fixed
At session start the build was RED. Three working-tree files were truncated
mid-file (content cut off at the tail), breaking tsc and the vitest config load:

- `client/src/pages/DevModeDiagnostics.tsx` — 57 lines missing (unclosed JSX).
- `server/pipelineService.ts` — tail of `processBookPipelineScenes` truncated
  (`'}' expected`).
- `vitest.config.ts` — cut off at `setupFiles` (`Expected "}"`), so vitest
  could not even load its config.

All three were restored from HEAD (which was intact) via `git show HEAD:<f> > <f>`.
The committed history was not affected; only the working copies were damaged.

Root cause is environmental, not a logic bug: every tracked file in the working
tree shows as modified due to CRLF line endings (HEAD is LF), and the three
truncations are the same class of corruption — a write path on the Windows mount
dropping file tails. I reproduced this live: editing a file through the
file-write tool truncated `ImageGalleryView.tsx` and `ImageGalleryVirtualized.tsx`
at the tail. Workaround used for the rest of the session: all writes done via
shell (`python`/heredoc), which write the full file correctly. **Flag for Jerome.**

## Addendum — Scenes-table cut-over (same day, after Jerome's approval)

Jerome approved Option A: a dedicated `scenes` table with a clean write-path
cut-over and no synthetic page rows. Implemented Phases 0–2.

**Schema (Phase 0).** Added `scenes` table (0-based `sceneIndex`, `title`,
`rationale`, `sourcePage`, `importance`, `description`, `prompt`,

page rows are created for scene-mode books. Failures upsert an error scene row
and call the new `markSceneForRetry`. Re-runs update existing scene rows rather
than duplicating (unique index enforces this).

(normalised to the page-shaped array the client renders, plus `sceneTitle` /
`sourcePage`) and reports `generationMode`. `shared/galleryImages.ts` gained a
dual path: real scene fields are used directly; legacy interim scene rows
(pre-cutover, metadata packed in ocrText) still render via the ocrText fallback;
page-mode rows fall back to "Page N".

**Decoupling invariant.** Upheld. The planner still returns prompts only; the
pipeline renders and records. OCR is never re-run in scene mode (explicit test).

**Tests.** Rewrote `scenePipeline.test.ts` for the scenes contract (6 tests,
incl. structured-capture + scene-mode flip + retry + idempotent re-run).
Extended `galleryImages.test.ts` (+3) for the real-field branch. Suite: 275
