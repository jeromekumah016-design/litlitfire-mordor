## New feature validated: render-side image-generation parameters

- `server/_core/imageParams.ts` — pure, no I/O:
  - `ImageGenParams { aspectRatio: square|portrait|landscape, quality: standard|hd, style: vivid|natural }`.
  - `DEFAULT_IMAGE_PARAMS` reproduces the legacy hardcoded `1024x1024` square/standard render, so books created before this change render identically.
  - `normalizeImageParams` coerces partial, malformed, or aliased input (`1:1`, `9:16`, `16:9`, pixel strings, `high`/`normal`, mixed case) into a complete valid object and never throws; unknown keys ignored; returns a fresh object (not the shared default).

- `generateImage` applies `size`/`quality`/`style` to the DALL-E 3 call. Offline path renders an aspect-correct SVG placeholder — exercises the full params path with zero spend.
- `processBookPipelineScenes(bookId, pdf, onProgress?, imageParams?)` resolves params once per book, threads them to every scene, and records the resolved render params in `scenes.generationParams` (alongside `style`/`mood`) for audit. No schema change.
